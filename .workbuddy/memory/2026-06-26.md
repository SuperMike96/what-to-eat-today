# 2026-06-26 工作日志

## 「今天吃什么」Web 应用 — 评审与修复

### 项目背景
React + TypeScript + Vite 的「今天吃什么」菜品滑动选择应用，用户已开发 MVP，需评审并修复。

### 关键问题与修复
1. **P0 入口错误**：`index.html` 引用 `/src/static-app.js`（原生 JS），精心写的 React 版 `App.tsx`/`main.tsx` 完全是死代码从未加载。修复为引用 `main.tsx`，删除 `static-app.js`。
2. **P0 注入风险**：`static-app.js` 用 `fetch` 拿 TS 源码 + `new Function` 动态执行，有注入风险且 `vite build` 后路径 404 必崩。已删除。
3. **PRD 缺失功能补齐**：
   - 桌面端键盘快捷键（← 跳过 / ↑ 待定 / → 想吃 / Ctrl+Z 撤销），用 `useRef` 持有最新 activeDishId 避免闭包旧值
   - 菜谱页补「食材处理」`ingredientSteps` + 「图文/视频占位」`media` 字段，20 道菜数据全补
   - 菜谱页左右滑动手势切换菜品
4. **体验优化**：拖拽回弹加 `cubic-bezier(0.16,1,0.3,1)` 过渡；history 加 50 条上限防 localStorage 膨胀；阈值 110/44 提取常量
5. **构建修复**：`tsc -b` 在 project references 模式下对 `@vitejs/plugin-react@4.7.0` 的 `declare function + namespace + export as default` 类型推断失配（`typeof viteReact` 不匹配 `PluginOption`）。改 `build` 为 `tsc --noEmit -p tsconfig.json`，只检查 src，跳过 vite.config.ts（社区主流做法）。

### 技术要点备忘
- `@vitejs/plugin-react` 默认导出的类型在 `moduleResolution: "Bundler"` + project references 下有摩擦，配置文件类型检查与源码分离更稳
- 拖拽组件用 `resting`/`dragging` 双状态控制 CSS transition：拖拽中 `transition: none` 跟手，松手回弹 `transition: transform 320ms`
- 键盘事件监听用 `useEffect` + `window.addEventListener`，依赖项 `[state.step, state.history.length]`
